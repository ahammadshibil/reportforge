# Code of Conduct

Be kind, be specific, ship things.

## What that means in practice

- **Be kind.** Critique code, never people. Default to assuming the other person is acting in good faith. If you've misread something, say so quickly.
- **Be specific.** "This doesn't work" gets ignored. "When I run X on env Y I get error Z" gets fixed.
- **Ship things.** Small PRs land faster than big ones. A working draft beats an unmerged perfect refactor.

## What we won't accept

- Personal attacks, harassment, demeaning language, dog-piling
- Off-topic political fights (BYOR is a reporting tool, not a forum)
- Promoting unrelated paid products in issues / PRs
- AI-generated noise PRs that don't actually solve a problem
